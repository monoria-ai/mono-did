# Security Policy

## Supported Versions

Security updates are provided for the latest published version line.

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Use a private reporting channel:

1. Open a private security advisory in the repository (preferred), or
2. Contact project maintainers directly via private communication.

Please include:

- A clear description of the issue
- Reproduction steps or proof of concept
- Affected package(s) and version(s)
- Suggested mitigation (if available)

## Response Expectations

- Initial acknowledgment: within 3 business days
- Triage and severity assessment: as soon as reproducible details are available
- Fix and coordinated disclosure timeline: shared after triage

## Disclosure

After a fix is available, maintainers will coordinate responsible disclosure
and publish relevant release notes.
